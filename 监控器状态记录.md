### 存在的问题

#### 主要问题：进度计算与GMGN平台不一致

**具体表现：**
- SQUISH代币：监控器显示79.7%，GMGN显示20.9%
- H4MGU4...代币：GMGN显示99.2%，监控器未出现
- 监控器中的最高进度仅为54%，远低于实际高进度代币

**可能原因分析：**
1. **API数据差异**：Pump.Fun API和GMGN API可能使用不同的数据源或计算方式
2. **数据延迟**：不同平台的更新频率和缓存机制不同
3. **计算逻辑差异**：进度计算公式可能存在细微差别
4. **API限制**：Pump.Fun API可能只返回部分代币，错过高进度代币

#### 技术细节

**当前进度计算逻辑：**
```python
def _extract_progress(self, item: Dict[str, Any]) -> float:
    if item.get('complete', False):
        return 1.0
    
    real_sol_reserves = item.get('real_sol_reserves', 0)
    virtual_sol_reserves = item.get('virtual_sol_reserves', 1)
    
    if virtual_sol_reserves <= 0:
        return 0.0
    
    progress = real_sol_reserves / virtual_sol_reserves
    return max(0.0, min(progress, 1.0))
```

**API获取策略：**
- 使用3种排序方式获取数据：`market_cap desc`、`created_timestamp asc`、默认排序
- 去重处理，合并所有获取的代币
- 只监控活跃代币（进度<100%）
